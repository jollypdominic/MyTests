🔴 Ceph Health Issues

116 issues (18.3%)

🔍 Root Causes

1. OSD Failures & Slow Operations

OSDs going down due to hardware failures, network issues, or resource exhaustion causing slow operations and degraded cluster performance.

2. Monitor Clock Skew (MON_CLOCK_SKEW)

Time synchronization issues between Ceph monitor nodes leading to quorum problems and cluster instability.

3. Network Heartbeat Delays

Public and cluster network delays causing OSD heartbeat failures, often due to network congestion or hardware issues.

4. PG Availability Issues

Placement group availability problems due to insufficient OSDs, replication failures, or cluster rebalancing.

✅ Prevention & Recommendations

1. Implement Proactive Monitoring
  • Set up automated health checks for OSD status every 5 minutes
  • Monitor disk SMART data and predict failures before they occur
  • Implement alerting for slow ops threshold (>30s)
  • Track network latency metrics continuously
2. Time Synchronization
  • Deploy NTP/Chrony on all Ceph nodes with redundant time sources
  • Set strict clock drift thresholds (max 50ms)
  • Monitor time sync status in real-time
3. Network Optimization
  • Separate public and cluster networks with dedicated NICs
  • Implement QoS policies for Ceph traffic
  • Use jumbo frames (MTU 9000) for cluster network
  • Regular network performance testing
4. Capacity Planning
  • Maintain at least 30% free capacity in clusters
  • Plan OSD additions before reaching 70% utilization
  • Implement automated rebalancing during maintenance windows

  • Add health-based routing to healthy backends
  • Maintain backup storage pools for failover

4. Libvirt Integration
  • Validate VM state before snapshot operations
  • Ensure qemu-guest-agent is running for consistent snapshots
  • Implement timeout mechanisms (max 10 minutes)
  • Add fallback to crash-consistent snapshots if needed
